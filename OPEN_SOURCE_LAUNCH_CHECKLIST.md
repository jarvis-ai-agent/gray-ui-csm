# Open Source Launch Checklist (X)

This checklist is tailored for `gray-ui-csm` before sharing on X as a design engineering portfolio project.

## 1) Pre-Launch Repo Checklist

- [ ] `pnpm install`
- [ ] `pnpm typecheck`
- [ ] `pnpm lint`
- [ ] `pnpm build`
- [ ] README reflects current architecture and key flows
- [ ] `LICENSE` exists
- [ ] `CONTRIBUTING.md` exists
- [ ] No private secrets or real customer data
- [ ] Demo data is clearly labeled as mock/demo

## 2) Portfolio Positioning

Use this framing in your post profile and comments:

- Design engineer focus: systems + UX details + implementation quality
- Product context: CSM workspace with real interaction depth
- What is reusable: app shell, data-grid patterns, ticket workflows

## 3) Asset Checklist for X Post

Prepare these assets before posting:

- [ ] 1 cover image (overall workspace)
- [ ] 1 short GIF for ticket board interactions
- [ ] 1 short GIF for ticket detail/reply flow
- [ ] 1 screenshot of code architecture (folder or helper extraction)
- [ ] Repo link in first post

## 4) Suggested Post Structure

Use a concise hook + proof + CTA:

1. Hook: what you built and for whom.
2. Proof: 2-3 concrete interaction details.
3. Engineering angle: architecture/refactor highlights.
4. CTA: ask for feedback, stars, or UX critique.
